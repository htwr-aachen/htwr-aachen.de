# Neue URL-Spezifikation

Ich will jetzt deutlich stabilere API und URL-Definitionen haben.

Noch sind wenige Fächer auf der neuen API, aber das wird sich schnell ändern:

- `/`: root HTWR Seite
- `/[institute]`: Die Oberseite eines Instituts/Lehrstuhls / Auswahl der Fächer
- `/[institute]/[subject]`: Die Seite eines Fachs. Individuelle Memes und Verlinkung zu:
- `/[institute]/[subject]/materials`: Materialien für ein Fach
- `/[institute]/[subject]/summaries`: Zusammenfassungen für ein Fach
- `/[institute]/[subject]/exams`: Klausurprotokolle für ein Fach

Zudem noch:

- `/panikzettel`: Übersicht über alle Panikzettel
- `/panikzettel/[subject].pdf`: Download eines Panikzettels für ein Fach
- `/blog`: Mein HTWR-Blog
- `/docs`: Ihr seid gerade da.
- `/...`: Alle möglichen kleinen/rechtlichen Sachen und eventuell ein paar Easter Eggs

## Veraltet (Deprecated)

Die neuen URL-Spezifikationen werden langsam umgesetzt. Ich versuche die alten Links funktional zu halten. (Keine Garantie)

- `/`: root HTWR Seite
- `/[institute]`: Die Seite eines Fachs. Memes und so
- `/[institute]/studium`: Die Seite eines Fachs. Individuelle Memes. Wurde manchmal übersprungen
- `/[institute]/studium/aufgaben`: Materialien für ein Fach
- `/[institute]/studium/teachings`: Zusammenfassungen für ein Fach und Folien-Merge und so...
